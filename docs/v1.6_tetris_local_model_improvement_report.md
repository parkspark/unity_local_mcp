# 로컬 모델 기반 Unity 테트리스 생성·검증 개선 보고서

작성일: 2026-07-17  
대상: `unity_local_mcp` + 로컬 Unity 프로젝트 `My project`

## 결론

로컬 `qwen3-coder:30b`가 Unity MCP를 통해 새 테트리스 씬과 스크립트를 생성했고,
반복 수정 끝에 **컴파일·런타임 오류 없이 자동 낙하와 피스 잠금/다음 피스 생성까지** 확인했다.
이 과정에서 외부 LLM API나 외부 Unity 제어 서비스는 사용하지 않았다.

최종 산출물:

- 씬: `C:\Users\park\My project\Assets\Scenes\LocalMcpTetrisV2.unity`
- 스크립트: `C:\Users\park\My project\Assets\Scripts\LocalMcpTetrisV2.cs`

다만 현재 검증은 자동 낙하 중심의 런타임 스모크 테스트다. 실제 키 입력,
회전/하드드롭, 줄 삭제와 점수 UI는 자동 입력으로 끝까지 검증하지 않았으므로
“기본 실행 확인”과 “완전한 게임성 검증”을 구분해야 한다.

## 로컬 전용 구성 확인

```text
qwen3-coder:30b (Ollama, localhost)
  -> unity_local_mcp Agent / TaskContract
  -> unity_mcp stdio server
  -> TCP 127.0.0.1 UnityMcpBridge
  -> Unity Editor
```

보조 시각 분석은 설치된 로컬 `qwen2.5vl:32b`를 사용할 수 있다. 자동 분석은
`UNITY_AGENT_AUTO_VISION=1`에서만 켜며, 32 GB VRAM 환경에서는 코더와 비전 모델을
순차적으로 운용하는 것이 안전하다.

## 생성·수정 과정

| 단계 | 관측된 결과 | 조치 |
|---|---|---|
| 1차 생성 | 컴파일은 성공했지만 회색 보드만 보이고 활성 피스가 보이지 않음 | 스크린샷을 근거로 실패 처리 |
| 2차 수정 | 피스 유형을 매 호출마다 무작위로 바꾸던 오류를 제거 | `currentPieceType` 고정, 잠긴 블록 시각 그리드 도입 |
| 3차 수정 | 활성 피스가 보드 좌표 `Y≈19`에 배치되어 카메라 밖에 있었음 | `BoardToWorld(x, y)` 좌표 변환을 사용하도록 수정 |
| 4차 수정 | 카메라/보드 렌더링이 빠져 조각이 잘려 보임 | 직교 카메라와 10×20 보드 배경 추가 |
| 최종 수정 | 보드가 활성 피스보다 카메라 앞에 있어 피스를 가림 | 보드 Z를 `+0.5`, 피스를 `0`으로 배치 |

이 사례는 “컴파일 오류 없음”이 게임 완성을 의미하지 않는다는 점을 분명히 보여준다.
실제 화면 검증이 없었다면 1차~4차 결과 모두 잘못 성공 처리됐을 가능성이 높다.

## 이번에 적용한 오케스트레이션 개선

### 1. TaskContract 정책 게이트

모델이 제안한 도구 호출을 그대로 실행하지 않고, 실행 전 정책 검사를 거치도록 했다.

- 범위 밖 기존 스크립트 읽기/삭제 차단
- `unity_execute_menu_item` 차단: 네이티브 저장 대화상자 방지
- 스크립트는 `Assets/Scripts/*.cs`로 제한
- 스크립트 검색은 `Assets/Scripts`, 최대 30개로 제한
- 수정 뒤 컴파일 검사, 플레이 뒤 대기/런타임 검사/정지를 완료 조건으로 강제
- 스크린샷을 요구한 작업은 실제 `unity_screenshot` 호출 전 완료 처리 금지

### 2. 대화상자 없는 씬 생성

`unity_create_scene(path, template)`을 추가했다. 이 도구는 새 씬을 지정 경로에 저장하며,
dirty 씬 폐기나 기존 씬 덮어쓰기를 거부한다.

```python
unity_create_scene(
    path="Assets/Scenes/LocalMcpTetrisV2.unity",
    template="basic",
)
```

### 3. 런타임 검증용 대기 도구

`unity_wait(seconds)`를 추가했다. 플레이 모드 직후 즉시 콘솔을 읽으면 `Start`/`Update`에서
발생하는 오류를 놓칠 수 있으므로, 0.5~10초 대기 후 콘솔을 검사한다.

```text
unity_play_mode(play)
  -> unity_wait(1)
  -> unity_read_console(error,exception)
  -> unity_screenshot(game)
  -> unity_play_mode(stop)
```

### 4. 발견·수정된 정책 버그

사용자 프롬프트에 두 개의 `Assets/...` 경로가 한 문장에 있을 때, 기존 경로 정규식이
둘을 하나의 긴 경로로 취급해 스크립트 읽기를 막는 문제가 있었다. 파일 확장자에서
경로 매칭을 끝내도록 수정하고 테스트를 추가했다.

## 남은 개선 권고

1. **입력 자동화 추가**: Unity Game View에 방향키, 회전, 하드드롭을 보내고 위치 변화를
   캡처하는 `unity_send_key` 또는 테스트 전용 입력 훅이 필요하다.
2. **결정적 줄 삭제 테스트**: 테스트 모드에서 거의 완성된 한 줄을 만들고 I 피스를
   하드드롭해 점수 증가와 행 이동을 자동 검증한다.
3. **시각 판정의 구조화**: 비전 모델의 자유 서술 대신 `board_visible`,
   `active_piece_visible`, `locked_piece_visible` 같은 JSON 판정 스키마를 사용한다.
4. **에이전트 완료 상태 개선**: 모델이 성공 후 불필요하게 `refresh`를 다시 호출하면
   이미 통과한 검증 상태가 초기화되어 반복될 수 있다. 완료 단계는 호스트 상태 기계가
   고정하고, 모델은 누락 단계만 수행하게 하는 편이 좋다.
5. **게임 UI 보강**: 현재 스크립트는 점수 값을 관리하지만 화면에 점수를 명확히 표시하지
   않는다. 다음 반복에서 카메라에 보이는 월드 공간 텍스트 또는 검증 가능한 UI를 추가한다.

```text
Assets/Scenes/MyTetris.unity에 새 테트리스 씬을 만들고,
Assets/Scripts/MyTetris.cs만 사용해 구현해 줘.
컴파일 오류 검사, 1초/10초 플레이 검사, 게임 스크린샷, 정지와 저장까지 수행해.
```

핵심은 모델의 최종 문장이 아니라, **Unity 콘솔·씬 상태·캡처 이미지**를 함께 통과 기준으로
사용하는 것이다.
